# Проект: Автоматизация E2E-сценария для Avito (Selenide)

Сложный End-to-End автотест для реального веб-ресурса (Avito), охватывающий полный цикл от авторизации до проверки глубоких элементов формы оформления заказа.

## 🎯 Сценарий теста
1. **Авторизация**: Вход в личный кабинет (настройка под тестовый контур).
2. **Умный поиск**: Переход в категорию "Личные вещи" (Сочи) с имитацией поискового запроса.
3. **Обработка динамических данных**: Циклический перебор объявлений (до 5 попыток) для поиска товара с доступной "Авито Доставкой".
4. **Работа с вкладками**: Переключение контекста драйвера между списком выдачи и карточкой товара.
5. **Валидация**: Проверка пустого значения поля "Телефон" на странице оформления заказа (Checkout).

## 🛠 Стек технологий
* **Java 11**
* **Selenide 6.x** — использование Fluent Interface и умных ожиданий.
* **JUnit 5** — современный движок для запуска тестов.
* **Maven** — управление зависимостями.

## 🚀 Ключевые особенности реализации
* **Fluent Interface (Method Chaining)**: Архитектура Page Object построена так, что шаги теста читаются как естественный текст. Методы возвращают объекты следующих страниц (`return page(NextPage.class)`).
* **Anti-bot Bypass**: Настроена маскировка WebDriver через `ChromeOptions` (`AutomationControlled`, `excludeSwitches`), что минимизирует риск блокировок со стороны антифрод-систем.
* **Стабильность (Fault Tolerance)**: Реализована обработка исключений (`try-catch`) внутри Page Object. Если в конкретном объявлении нет кнопки доставки, тест не падает, а закрывает вкладку и переходит к следующему лоту.
* **Smart Waits**: Вместо жестких пауз используются динамические ожидания Selenide (`shouldBe(visible)`) с кастомными таймаутами для тяжелых страниц.

## 📂 Структура проекта
* `pageobjects/` — реализация паттерна Page Object (MainPage, LoginModal, SearchResultPage, ItemPage, Checkout).
* `Tests.java` — основной тестовый класс с логикой E2E-сценария.

---
**Важно:** В соответствии с требованиями задания, логин и пароль в коде оставлены пустыми (`""`). Тест подготовлен для запуска на специализированном тестовом стенде.
