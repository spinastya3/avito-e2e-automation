# 🚀 Личный Pet-проект: Автоматизация E2E-сценария для Avito — <u>Selenide</u>

**Статус проекта:** `Самостоятельная разработка (Pet Project для развития навыков QA Automation)`

Сложный End-to-End автотест для реального веб-ресурса (Avito) с высоким уровнем защиты от ботов. Проект разработан полностью с нуля для отработки навыков автоматизации на сложном динамическом фронтенде, охватывая полный цикл от авторизации до проверки глубоких элементов формы оформления заказа.

---

### 🎯 Сценарий теста
1. **Авторизация:** Вход в личный кабинет (настройка под тестовый контур).
2. **Умный поиск:** Переход в категорию "Личные вещи" (г. Сочи) с имитацией поискового запроса.
3. **Обработка динамических данных:** Циклический перебор объявлений (до 5 попыток) для поиска товара с доступной "Авито Доставкой".
4. **Работа с вкладками:** Переключение контекста WebDriver между списком выдачи и новой карточкой товара.
5. **Валидация:** Проверка пустого значения поля "Телефон" на странице оформления заказа (Checkout).

### 🚀 Ключевые особенности реализации
*   **Самостоятельное проектирование:** Сценарии, локаторы и архитектура тестов определены без учебных ТЗ или подсказок, на основе анализа реального продакшн-сайта.
*   **Fluent Interface (Method Chaining):** Архитектура Page Object построена так, что шаги теста читаются как естественный текст. Методы возвращают объекты следующих страниц через `return page(NextPage.class)`.
*   **Anti-bot Bypass (Обход защиты Avito):** Настроена маскировка WebDriver через конфигурацию `ChromeOptions` (`AutomationControlled`, `excludeSwitches`), что минимизирует риск блокировок со стороны антифрод-систем реального ресурса.
*   **Стабильность (Fault Tolerance):** Реализована обработка исключений (`try-catch`) внутри бизнес-логики страниц. Если в конкретном объявлении нет кнопки доставки, тест не падает, а корректно закрывает вкладку и переходит к следующему лоту в цикле.
*   **Smart Waits:** Вместо жестких пауз (Thread.sleep) используются встроенные динамические ожидания Selenide (`$.shouldBe(visible)`) с кастомными таймаутами для тяжелых элементов интерфейса.

### 💻 Запуск автотестов
> **Важно:** В соответствии с требованиями безопасности, личные логин и пароль в коде оставлены пустыми `""`. Перед запуском необходимо указать тестовые данные.

Для локального запуска тестов выполните команду в терминале:
```bash
mvn clean test
```
